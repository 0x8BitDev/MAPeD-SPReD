Graphics Banks Data




Each graphics bank contains: CHR bank (1x1), blocks (2x2), tiles (4x4), palettes and screens data.




Int32 mpd_num_banks()



Get the number of graphics banks in a project.


Returns:

the number of graphics banks in a project




Int32 mpd_get_active_bank()



Get an active graphics bank index.


Returns:

the index of an active bank




Int32 mpd_get_tiles_cnt( Int32 _bank_ind )



Get the number of used tiles (4x4) in a graphics bank.


Parameters:

_bank_ind – graphics bank index


Returns:

the number of used tiles or -1 in case of incorrect bank index




Int32 mpd_get_blocks_cnt( Int32 _bank_ind )



Get the number of used blocks (2x2) in a graphics bank.


Parameters:

_bank_ind – graphics bank index


Returns:

the number of used blocks or -1 in case of incorrect bank index




Int32 mpd_get_CHRs_cnt( Int32 _bank_ind )



Get the number of used CHRs (1x1) in a graphics bank.


Parameters:

_bank_ind – graphics bank index


Returns:

the number of used CHRs or -1 in case of incorrect bank index




Array[UInt64] mpd_get_tiles( Int32 _bank_ind )



Get an array of 4x4 tiles in a graphics bank.


Parameters:

_bank_ind – graphics bank index


Returns:

the 4x4 tiles array or null in case of incorrect bank index


Each tile value (UInt64) consists of block's indices ordered left to right, top to bottom from high to low word




Array[UInt32] mpd_get_blocks( Int32 _bank_ind )



Get an array of blocks ( 2x2 tiles ) in a graphics bank.


Each block's CHR data (UInt32) has the following bits description:

NES: 15.. [ property_id ](4) [ palette ind ](2) [X](2) [ CHR ind ](8) ..0

SMS: 15.. [ property_id ](4) [ hv_flip ](2) [ palette ind ](1) [CHR ind](9) ..0

PCE: 19.. [ property_id ](4) [ palette ind ](4) [CHR ind](12) ..0

ZX: 15.. [ property_id ](4) [ palette ind ](1) [CHR ind](11) ..0

SMD: 19.. [ property_id ](4) [x](1) [ palette ind ](2) [ hv_flip ](2) [CHR ind](11) ..0


Four UInt32 values form one 2x2 tile (block).


Parameters:

_bank_ind – graphics bank index


Returns:

the blocks data array or null in case of incorrect bank index




Array[Byte] mpd_get_CHRs_data( Int32 _bank_ind )



Get a raw CHRs data. The CHR data is stored as 128x128 x number of pages image (8bit).


Parameters:

_bank_ind – graphics bank index


Returns:

the array of bytes or null in case of incorrect bank index




Array[Byte] mpd_get_CHR( Int32 _bank_ind, Int32 _CHR_ind )



Get a raw CHR data by index. The data array length is 64 bytes (8x8 pixels).


Parameters:

_bank_ind – graphics bank index

_CHR_ind – CHR index ( min = 0; max = ( mpd_get_CHRs_data( bank_n ).Count / 64 ) - 1 )


Returns:

the array of bytes or null in case of incorrect bank index




NES: Long mpd_export_CHR_data( Int32 _bank_ind, String _filename, Bool _save_padding )

SMS: Long mpd_export_CHR_data( Int32 _bank_ind, String _filename, Int32 _bpp )

PCE/ZX/SMD: Long mpd_export_CHR_data( Int32 _bank_ind, String _filename )



Convert a CHR bank into platform compatible format and save it to a file.


Parameters:

_bank_ind – graphics bank index

_filename – exported data filename

_save_padding – save padding data aligned to 1/2/4 KB or not

_bpp – bits per pixel value ( the valid range is 1-4 )


Returns:

the size of a saved file in bytes or -1 in case of an error

Returns:

the array of 4x4 or 2x2 tiles indices of a particular screen or null in case of incorrect bank index



Layouts Data




Each layout is a matrix of cells. Each cell can contains a screen index and marks ( adjacent screens mask + screen property value 0...15 ).




Int32 mpd_num_layouts()



Get the number of layouts available in a project.


Returns:

the number of layouts




Int32 mpd_layout_width( Int32 _layout_ind )



Get a layout width.


Parameters:

_layout_ind – layout index


Returns:

the input layout width or -1 in case of incorrect layout index




Int32 mpd_layout_height( Int32 _layout_ind )



Get a layout height.


Parameters:

_layout_ind – layout index


Returns:

the input layout height or -1 in case of incorrect layout index




Int32 mpd_layout_start_screen_cell( Int32 _layout_ind )



Get a cell index which contains a start screen index.


Parameters:

_layout_ind – layout index


Returns:

the cell index or -1 in case of incorrect layout index


To get a start screen index you can do the following steps:


layout_width = mpd_layout_width( layout_n )

layout_height = mpd_layout_height( layout_n )

start_screen_cell = mpd_layout_start_screen_cell( layout_n )

start_screen_ind = mpd_layout_screen_ind( layout_n, start_screen_cell % layout_width, start_screen_cell / layout_width ) )






Int32 mpd_layout_screen_ind( Int32 _layout_ind, Int32 _scr_pos_x, Int32 _scr_pos_y )



Get a screen index by cell coordinates ( zero-based values ).


Parameters:

_layout_ind – layout index

_scr_pos_x – cell index by X axis

_scr_pos_y – cell index by Y axis


Returns:

the screen index in a common screens array of all screens of all graphics banks ( in the editor you can set the 'All Banks' checkbox ( Layout tab → Screens tab ) to see the common screens array ) or -1 in case of incorrect layout index




UInt16 mpd_layout_screen_marks( Int32 _layout_ind, Int32 _scr_pos_x, Int32_scr_pos_y )



Get screen marks assigned to a screen with particular coordinates ( zero-based values ).


Parameters:

_layout_ind – layout index

_scr_pos_x – cell index by X axis

_scr_pos_y – cell index by Y axis


Returns:

the screen marks value: 7-4 - bit mask of user defined adjacent screens ( Down(7)-Right(6)-Up(5)-Left(4) ); 3-0 - screen property

or 0xFFFF in case of incorrect layout index



Entity Instances




Entity instances inherit all data of base entities and they are stored in screen data.




Entity instance class:



class mpd_inst_entity

{

// autogenerated unique id

public Int32 uid;


// unique id of a target entity

public Int32 target_uid;


// instance coordinates

public Int32 x;

public Int32 y;


// base entity id

public Int32 base_ent_uid;


// instance properties

// space separated decimal values

public String properties;

}




Int32 mpd_layout_screen_num_entities( Int32 _layout_ind, Int32 _scr_pos_x, Int32 _scr_pos_y )



Get the number of entities in a screen with particular coordinates.


Parameters:

_layout_ind – layout index

_scr_pos_x – cell index by X axis

_scr_pos_y – cell index by Y axis


Returns:

the number of entities or -1 in case of incorrect layout index




mpd_inst_entity mpd_layout_get_inst_entity( Int32 _layout_ind, Int32 _scr_pos_x, Int32 _scr_pos_y, Int32 _ent_ind )



Get an entity instance data.


Parameters:

_layout_ind – layout index

_scr_pos_x – cell index by X axis

_scr_pos_y – cell index by Y axis

_ent_ind – entity index


Returns:

the mpd_inst_entity object or null in case of incorrect layout index



Base Entities




Base entity class:



class mpd_base_entity

{

// user defined unique id

public Byte uid;


// entity dimensions

public Byte width;

public Byte height;


// origin offset

public SByte pivot_x;

public SByte pivot_y;


// string name

public String name;


// base properties inherit by all instances

// space separated decimal values

public String properties;


// instance properties by default (can be overridden by each instance)

// space separated decimal values

public String inst_properties;

}
